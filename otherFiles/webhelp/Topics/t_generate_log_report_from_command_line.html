Generate Log Report from Command Line

LexLogc or VLLogc can be used from the command line to generate a report that is either stored in an HTML file, or e-mailed to a list of users.

This feature allows a user to schedule a report to be automatically generated at a certain time.

Usage of command line LexLogc/VLLogc is as follows:

usage: LexLogc/VLLogc [-f xmlFilterFile] [-d file or e-mail] [-t title]

-?
Displays the command line options
-f "xmlFilterFile"
Uses the file specified by "xmlFilterFile" to set the Log Report Filter settings. A filter can be saved using Save As… in the View the Log File dialog. If the –f option is not present on the command line, only the current day's log will be in the report. 
-d "file"
–d "e-mail address(es)"
Outputs the log report (in an HTML format) to the listed "file" or "e-mail address(es)".  If the –d parameter contains an @ symbol, the parameter is assumed to be an e-mail address. Commas should separate multiple e-mail addresses. If the –d option is not present, the results will be printed to the console.
-t "title "
The –t parameter is appended to the HTML report title and the email subject if applicable. If the desired title contains whitespace, enclose the parameter within double-quotes ("...").
