Transfer Details Table

The Transfer Details table provides details of every transfer bound by the current filter (see Transfer Dashboard Filter Area) and parameter settings (see Transfer Dashboard Parameters). The table has two grouping levels; first by trading partner and next by the host\mailbox. The following summary calculations can be seen at each group level:

  • Total Successful Transfers (those with a General Status of Successful)
  • Total Failed Transfers (those with a General Status of Failed)
  • Total In-Progress Transfers (those with a General Status of In Progress)
  • Percentage Successful Transfers (total Successful transfers out of total transfers)

Initially the table is sorted by Trading Partner, then by Host\Mailbox, and finally Start Time. See Manipulating Dashboard Components for information about how to use the table's right-click menus. 

In the upper right corner of the table, a Total Records value is displayed:

Total Records provides the number of records displayed in the Transfer Details table as well as the number of records represented in the Transfer Summary chart above. Initially, it will represent the number of records in the query; however, it will change if any filters are selected. 

Below is a description of each column in the Transfer Details table:

  • Start Time specifies the start time for the transfer. This is a linked cell. Clicking on a specific start time will display the Transfer Detail Report. See Transfer Detail Report.
  • End Time specifies the end time for the transfer. This field will be blank for a transfer that is in progress.
  • Status specifies the specific status of the transfer.
  • Direction specifies whether the transfer is outbound (send) or inbound (receive).
  • Protocol specifies the common name for the transport mechanism.
  • File Name specifies the file name.
  • File Type specifies the last extension of the file name (in lowercase).
  • File Size specifies the file size in bytes.
  • Tracked Type specifies whether there is associated tracked data for the transfer (EDI, XML, or TEXT). If no data is associated, it will report untracked.
  • Run Type specifies how the transfer was run (Interactive, Scheduled, API, etc.).
  • More provides a link to possible features that are available for the associated transfer.  At this time, More links directly to a file viewer. File viewing is only available if certain conditions were met at the time of the transfer. Contact your VersaLex system administrator to ensure the following conditions are met to enable the viewing capability:
    • For remote hosts, the sentbox and receivedbox are configured.
    • For local hosts, the archive sentbox and receivedbox are configured.
    • Configure > Options > Other > Disable Date/Time Portion Of Filenames In Sent/Receivedbox is cleared.
    • When you click More, if the above conditions were not met, and therefore a copy of the file is not available, you will receive an error.

      If, however, a copy of the file is available, the File Viewer will display the file. See Transfer Dashboard File Viewer.
