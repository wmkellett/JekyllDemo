Cleo Dashboards and System Monitors Overview

The Cleo Dashboards™ and Cleo System Monitor™ packages are web applications that provide a user portal interface for the business users (Cleo Dashboards) and the system administrators (Cleo System Monitor). You can view any of the preconfigured dashboards or you can customize your viewing experience with a broad range of possibilities.  

Note Regarding "Dashboard" Terminology:

The Cleo software discussed in this chapter are formally referred to as the Cleo Dashboards™ and the Cleo System Monitor™ products. Within both of these products, there is an object referred called the dashboard. A dashboard is a type of report; specifically, it is a workspace window that consists of library components. 

Dashboard and System Monitor Concepts

The concepts below are key elements of the Cleo Dashboards™ and Cleo System Monitor™ products:

Library component
Library components are used to build dashboards. They are able to present data via components such as charts, gauges, and tables. Many pre-built library components are built and provided by Cleo, but custom library components can be provided. All library components are suffixed with .lc.
Report
A report is an object that can be displayed within a browser window or tab. Reports are further broken into web reports and dashboards.
Web report
Web reports are generally used to communicate a single type of component, whether it is a chart or a table. Typically web reports are used to communicate detailed, drill-down data. Web reports are often presented as sub-reports of a library component or another web report. All web reports are suffixed with .wls.
Dashboard
A workspace window that can contain any number of library components. There are several resources (see Resources Panel) that you can insert into the dashboard; the most common resources are library components and filters. All dashboards are suffixed with .dsh.
