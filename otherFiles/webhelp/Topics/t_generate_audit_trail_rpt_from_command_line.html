Generate Operator Audit Trail Report from Command Line

Note: This section applies to the Cleo Harmony and Cleo VLTrader applications only.

VLTraderc/Harmonyc can be used from the command line to generate an Operator Audit Trail report that is either:

  • Stored in an HTML file
  • Stored in a CSV file
  • E-mailed to a list of users

Using this feature, the user can use operating system features to schedule a report to be automatically generated at a certain time.

Usage of command line VLTraderc/VLHarmonyc is as follows:

VLTraderc/VLHarmonyc –s OpAudit [-f xmlFilterFile] [-d file or e-mail] [-t title]

-s OpAudit: Generates an Operator Audit Trail report.

-f "xmlFilterFile": Uses the file specified by "xmlFilterFile" to set the Operator Audit Trail Filter settings. A filter can be saved using Save As in the Operator Audit Trail Filter dialog box. See Operator Audit Trail. If the –f option is not present on the command line, only the current days operator audit trail events will be in the report.

-d "file" or –d "e-mail address(es)" : Outputs the Operator Audit Trail report (in an HTML format) to the listed "file" or "e-mail address(es)". If the filename has a .csv extension, then the file will be output as a comma-separated values (CSV) formatted file. If the –d parameter contains an @ symbol, the parameter is assumed to be an e-mail address. Commas should separate multiple e-mail addresses. If the –d option is not present, the results will be displayed in CSV format to the console.

-t "title": The –t parameter is appended to the HTML report title and the email subject if applicable. If the desired title contains whitespace, enclose the parameter within double-quotes ("...").
