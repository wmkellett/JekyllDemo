Transfer Log

The following is a description of the fields used for the transfer log database feature.  The same fields appear in the database columns and the XML file depending on how you wish to store the data.

VLTransfers database table or logs/xferYYYYMMDD.xml XML file
Column/Field Name Data Type Length Description
TransferID VARCHAR 30 Transfer ID (generated by VersaLex and used as a "key" to the records)
ExternalID

Database only

 VARCHAR 50 Optional external ID provided by end user application
MessageID VARCHAR 100 Message ID (generated either by VersaLex or the remote host)
Folder

Database only

 VARCHAR 50 Host folder alias
Host VARCHAR 50 Host Alias
Mailbox VARCHAR 50 Mailbox Alias
MailboxID

Database only

VARCHAR 255 Optional identifier set via mailbox externalID property.
Username

Database only

VARCHAR 50 Name of user performing the transfer. If it matches the Mailbox, then this is null.
Action VARCHAR 50 Action Alias
Transport VARCHAR 12 Transport
StartDT VARCHAR 20

Start Date and Time the file transfer started

(Format: yyyy/mm/dd hh:mm:ss)
EndDT VARCHAR 20

End Date and Time the file transfer completed

(Format: yyyy/mm/dd hh:mm:ss)
Direction VARCHAR 10 Direction ("send" or "receive", or "inbound" or "outbound" for CHECK commands)
IsReceipt VARCHAR 1 Receipt file ("T"=True or "F"=False)
Status VARCHAR 15

Status (“In Progress”, “Receipt Pending”, "Interim Success"*, “Interim Warning”*, "Delete Error", "Delete Resolved", “Success”, “Warning”, “Discarded”, “Error”, “Exception”, or “Interrupted”)

* (Database payload only)

OrigName VARCHAR 100 Original filename (if file is a zip archive, then this field contains "zip filename[entry name]")
OrigPath VARCHAR 200 Original path
OrigFileDT VARCHAR 20

Original file date/time stamp

(Format: yyyy/mm/dd hh:mm:ss)
FileSize BIGINT   File size
TransferTime FLOAT 20 Transfer time in seconds
TransferBytes BIGINT   Transfer bytes
CRC

Database only

 VARCHAR 12 CRC-32 value associated with the transfer, or NULL if CRC is not available
ResultText VARCHAR 500 Result text
FileHeader VARCHAR 100 File Header (future use)
VLSerial

Database only

VARCHAR 6 VersaLex license serial number (in case multiple VersaLexes sharing database)
CopyPath

Database only

VARCHAR 500 Contains the path to the sentbox/receivedbox copy or NULL if a copy is not available.
RunType

Database only

VARCHAR 30 Contains the run type (e.g., "Interactive", "Scheduled", "Unsolicited", etc.) or NULL if the run type is not available.
PreviousTransferID

Database only

VARCHAR 30 If RunType = "Resend", contains the transfer ID of the transfer from which this transfer was based.  Is NULL otherwise.
Command

Database only

VARCHAR 500 The original command string.  Only present for CHECK commands.
InteractiveUsername

Database only

VARCHAR 50 Name of user running the action interactively.
StartNDT

Database only

DATETIME   DateTime version of StartDT
EndNDT

Database only

DATETIME   DateTime version of EndDT
TradingPartnerAlias

Database only

VARCHAR 255 Alias of trading partner associated to the host/mailbox or to the ID if tracking is enabled.
FileType

Database only

VARCHAR 255 File extension of transferred file
TrackedType

Database only

VARCHAR 25 If the file is tracked, then this will be the tracked type (“EDI”, “TEXT”, or “XML”).
IPAddress

Database only

VARCHAR 45 IP Address of remote computer used for local user IP filtering.
StartDTInt BIGINT   Milliseconds epoch version of StartDT
EndDTInt BIGINT   Milliseconds epoch version of EndDT

* A status of "Interim Success" when using database payload indicates that the VersaLex FTP or SSH FTP server received a file, but it was not stored by the FTP user into the user's configured inbox subdirectory.  Another transfer will be logged with a status of "Success" if and when the file is moved into the inbox directory by the FTP user via FTP rename commands.  (In fact, when the file is renamed by the FTP user, the payload is actually inserted into the VLIncoming database table at that time.)  See below for more information on the database payload feature.
