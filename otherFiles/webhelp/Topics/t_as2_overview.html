AS2 Overview 

The generic AS2 Host and Local Listener are provided to allow a user to fully specify, from scratch, an AS2 installation. If at all possible, use a pre-configured host specific to the remote server and a Local Listener specific to the local server. This makes setting up your installation faster and easier. See Local Listener.

The AS2 standard provides the ability to securely transport EDI (and other data, including binary and XML) to a remote host, guaranteeing that the message has not been changed in-transit and is received and can be read only by the intended trading partner. An MDN receipt (Message Disposition Notification) further guarantees that the intended trading partner has received the message.

AS2 uses the HTTP protocol as its transport mechanism to send files over the Internet. VersaLex uses the PUT (HTTP POST) action command to transport the secure data to the remote host.

VersaLex supports AS2 versions 1.0, 1.1 and 1.2.
