SLA/KPI Details Table

The SLA/KPI Details table provides details of every checkpoint bound by the current filter (see SLA/KPI Dashboard Filter Area) and parameter settings (seeSLA/KPI Dashboard Parameters). The table has two grouping levels: first by trading partner, and next by the host\mailbox. The following summary calculations can be seen at each group level:

  • Total Successful Checkpoints (those with a General Status of Successful)
  • Total Failed Checkpoints (those with a General Status of Failed)
  • Percentage Successful Checkpoints (total Successful checkpoints/total checkpoints)

The table is initially sorted by Trading Partner, then by Host\Mailbox, and finally by Run Time. See Manipulating Dashboard Components for general information about how to use the table's right-click menus. 

In the upper right corner of the table, a Total Records value is displayed.

Total Records provides the number of records displayed in the SLA/KPI Details table as well as the number of records represented in the SLA/KPI Summary chart above. It will initially represent the number of records in the query; however, it will change if any of the filters are selected. 

The columns in the SLA/KPI Details table are:

  • Run Time: specifies the time that the CHECK command was run. This is a linked cell.  Clicking on a specific run time will display the SLA/KPI Detail Report. See SLA/KPI Detail Report.
  • Check Type: specifies the type of CHECK command run. The types are:
    • Transfer
    • File Age
    • Directory Age
  • Class: specifies the setting of the Class parameter on the CHECK command.
  • Subclass: specifies the setting of the Subclass parameter on the CHECK command.
  • Status: specifies the specific status of the checkpoint.
  • Time Condition: specifies the time condition, as interpreted from the CHECK command's OccurredWithin/Age parameters.
  • Direction: specifies whether the CHECK was for outbound or inbound transfers (the value will be n/a for file and directory age checkpoints).
  • Path: specifies the path setting on the CHECK command.
