VLProxy Dashboard

The VLProxy Dashboard provides system information related to the VLProxies configured in the currently selected VersaLex, which can be changed through the top bar.

VLProxy Active Connections Table

The VLProxy Active Connections table displays each of the configured VLProxy URLs, when the last communication with the VLProxy occurred, and number of active connections to the VLProxy. If the Last Communication was over three minutes ago, then the table entry will turn red.

If there are any red entries, that VLProxy is not communicating with this VersaLex. The Last Communication time indicates when it last successfully communicated. The number of connections can be monitored against normal levels.

Access to the VLProxy Connection Count History sub-report is available by clicking on the underlined value in the table.

VLProxy Connection Count History Report

The VLProxy Connection Count History report displays the number of connections active in VLProxy over time. By default, it will show the last 60 minutes of history. Limit the history displayed to one or more VLProxies by selecting them in the VLProxy filter. Change the Start date/time of the chart and the number of Minutes displayed using the parameter control box. Use the zoom slider at the bottom to zoom in closer to a specific time. Holding the cursor over a specific data point on the chart will display information about that data point.

VLProxy CPU Usage Chart

The VLProxy CPU Usage chart displays the CPU usage of the VLProxy.

VLProxy does not typically use much CPU. A continuously high CPU load can indicate an issue; another VLProxy should be added to the system to spread out the load.

Access the VLProxy CPU Usage History sub-report by clicking on a needle in the graph.

VLProxy CPU Usage History Report

The VLProxy CPU Usage History report displays the CPU load as a percentage of total CPU capacity over time. By default, it will show the last 60 minutes of history. Using the VLProxy filter, you can limit the history displayed to one or more VLProxies. Use the parameter control box to change the Start date/time of the chart and the number of Minutes displayed. Use the zoom slider at the bottom to zoom in closer to a specific time. Holding the cursor over a specific data point on the chart will display information about that data point.

VLProxy Heap Memory Chart

The VLProxy Heap Memory chart displays the percentage of the configured heap memory used by the VLProxy.

Relationships (such as AS2-To / AS2-From), usernames, passwords, certificates, and many other items are stored in memory for use by VLProxy for each of the VersaLexes connected. Heap memory is also used to proxy data between the VersaLexes and trading partners. If the heap memory is nearing 100%, then the maximum heap memory should be increased. If this is due to proxying data and the maximum heap memory cannot be increased, this indicates that another VLProxy should be added to spread out the load.

Access the VLProxy Heap Memory History sub-report by clicking on a bar or a bar label.

VLProxy Perm Gen Memory Chart

The VLProxy Perm Gen Memory chart displays the percentage of the perm gen memory usage by VLProxy.

If this is nearing 100%, then the maximum perm gen memory should be increased.

Access the VLProxy Perm Gen Memory History sub-report by clicking on a bar or a bar label.

VLProxy Thread Count Chart

The VLProxy Thread Count chart displays counts of the active and deadlocked threads in VLProxy. It also displays the peak number of threads used by VLProxy since it was started.

Any deadlocked threads are indication of a problem, and a thread dump should be obtained from VLProxy and sent to Cleo Support. Otherwise, the number of threads depends on the amount of proxying occurring. An abnormally high number of threads can indicate an issue where trading partner connections are not being closed.

Access the VLProxy Thread Count History sub-report by clicking on a bar or a bar label.
