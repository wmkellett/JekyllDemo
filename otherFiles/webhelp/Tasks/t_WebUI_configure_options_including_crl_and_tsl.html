Configuring Options including CRL and TSL

The following describes how to configured certificate manager options.

  1. In the web UI, go to Administration > Certificate Management > Certificates. In the native UI, go to Tools > Certificate Manager or click the Certificates button in the tool bar.
    The Certificate Manager Options dialog box appears.
  2. Select from the following options and click OK.
    • Check certificate validity period will color certificates that are expired or within the warning period (15 days) red and orange, respectively.
    • Check certificate issuer’s CA signature includes verification of the issuer’s signature when building the certificate chain.
    • Check certificate verification checks the validity of the certificate’s signature algorithm and that it can be used within the current environment. When in FIPS mode, this setting is on by default and cannot be disabled. When not in FIPS mode, this setting defaults to off. If a certificate fails the verification check, it is marked with .
    • Check Revoked Certificates checks each user and CA certificate’s revoked status every [n] hours by using either the certificate’s OCSP (Online Certificate Status Protocol) URL or CRL (Certificate Revocation List) URL. If a revoked certificate is found, it is marked with  and cannot be used during file transfers, whether as a server or client certificate, for signing or encryption, and so on. If it is a CA certificate, its issued certificates also cannot be used. Besides checking CRLs actually contained within the certificates in the store, additional CRL URLs provided by certificate authorities can be configured and checked, if necessary. 
    • Import Trusted Service (Status) List causes the configured TSL URLs to be downloaded and imported every [n] hours. A TSL contains a set of CA certificates to be automatically trusted. Whether a CA certificate is added or removed from the TSL, it is likewise added or removed from the local certificate store.  Click Import Now to start a new import in the background.
    • Logging can be Enabled to capture Low, Medium, or High levels of certificate debug. 'High' is recommended while debugging a problem. The certificate manager debug log file can be found under the home directory at 'logs\CertMgrLogfile.txt'. It contains information relative to security providers, certificate parsing, chaining, and usage, and UI invocation. Since the debug file will continue to grow, certificate debug logging should only be turned on while investigating an issue, and should be turned back off once the investigation is complete.
  3. Click View Results… to see the status of the certificate revocation checks.
    Each OCSP and CRL URL is listed along with the status result.  Possible results are:
    • No revoked certificates found
    • Revoked certificate(s) found
    • Check error: …
    A check error can occur for varying reasons such as the URL being unreachable or the site returning an HTTP error code. Click Check Now to cause a new check to start in the background. Click Refresh to update the display if a check has just finished in the background.
