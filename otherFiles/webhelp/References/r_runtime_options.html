Runtime Options

Following table lists five different Cleo Harmony runtime options and when each is used:

(1) Run action via Cleo Harmony UI

(2) Install the Cleo Harmony application as a Windows service or run Cleo Harmony as a Unix daemon. See Run as a Unix Daemon.

(3) Run action via Cleo Harmony command line.

See Execute from Command Line.

(4) Map /mount Cleo Harmony installed drive and run action remotely via Cleo Harmony command line. See Execute from Command Line.

(5) Run action remotely via Cleo Harmony autorun. See Use Autorun.
If manually run actions  or      
If schedule actions within the Cleo Harmony application to run either periodically or whenever there is a file to send  or      
If using Cleo Harmony application as an "always live" server to receive files (e.g. AS2)  or      
If 3rd party software (e.g. translator) runs actions        
If a remote computer needs to run an action        or

(1), (2), and (3) are not mutually exclusive.  If the Cleo Harmony application is installed and/or running as a service/daemon, the Cleo Harmony UI and command line will execute against the running service/daemon.  But note:

If the Cleo Harmony application is installed as a service on Windows, by default, windows services run under a SYSTEM user and do not see mapped drives.  Use full network path names for the directories and ensure proper user authorization.  If necessary, change the service to log on under a different account.

(4) requires (2).  In order to run an action remotely via command line, the Cleo Harmony application must be installed and/or running as a service/daemon on the target computer.

(5) requires (1) or (2).  In order to run an action remotely via autorun, the Cleo Harmony UI must be running or the Cleo Harmony application must be installed and/or running as a service/daemon on the target computer.
